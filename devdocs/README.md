# Developer Documentation

This folder contains developer documentation for the Logic Service microservice, part of the Buildly RAD Core architecture.

## Documentation Structure

- **architecture.md** - Overview of the microservice architecture and design patterns
- **api-guidelines.md** - REST API development standards and best practices
- **development-setup.md** - Local development environment setup and workflow
- **testing-guide.md** - Testing strategies, patterns, and requirements
- **deployment.md** - Deployment processes and environment configuration
- **contributing.md** - Development workflow and contribution guidelines

## Quick Start

1. Read the [development-setup.md](./development-setup.md) for local environment setup
2. Review [architecture.md](./architecture.md) to understand the service design
3. Follow [api-guidelines.md](./api-guidelines.md) when developing new endpoints
4. Implement tests according to [testing-guide.md](./testing-guide.md)

## Buildly Standards

This service follows Buildly microservice standards. For AI developers, see the prompts in [.github/prompts/](../.github/prompts/) for detailed guidelines on maintaining consistency with Buildly architecture patterns.

## Updates and Changes

This documentation should be updated whenever:
- New architectural patterns are introduced
- API standards change
- Development workflow is modified
- Deployment processes are updated
- Testing requirements evolve
